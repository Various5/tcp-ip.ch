HTTP/HTTPS
80/443
HyperText Transfer Protocol is the foundation of data communication on the World Wide Web. HTTPS adds security through SSL/TLS encryption, protecting data in transit.
  • Stateless request-response protocol
  • Supports multiple request methods (GET, POST, PUT, DELETE)
  • Header-based metadata transmission
  • Content negotiation and caching support
  • HTTP/2 and HTTP/3 improvements for performance
Security Considerations:
Always use HTTPS for sensitive data. Implement proper authentication, validate all inputs, and use security headers to prevent common attacks.
WebSocket
80/443
Provides full-duplex communication channels over a single TCP connection. Essential for real-time applications like chat, gaming, and live data feeds.
  • Persistent bidirectional connection
  • Low latency real-time communication
  • Built on top of HTTP upgrade mechanism
  • Binary and text message support
  • Automatic connection management
Security Considerations:
Use WSS (WebSocket Secure) for encryption. Implement proper origin validation and rate limiting to prevent abuse.
HTTP/3 (QUIC)
443
The latest HTTP protocol built on QUIC (Quick UDP Internet Connections). Provides improved performance, reduced latency, and better connection resilience.
  • UDP-based transport for reduced latency
  • Built-in encryption and security
  • Multiplexed streams without head-of-line blocking
  • Connection migration support
  • 0-RTT connection establishment
Security Considerations:
QUIC has security built-in by design. Monitor for proper certificate validation and ensure firewall rules accommodate UDP traffic.
SMTP
25/587/465
Simple Mail Transfer Protocol handles the sending of email messages between servers and from email clients to servers. Essential for email delivery infrastructure.
  • Text-based command/response protocol
  • Relay capability for message routing
  • Support for authentication (SMTP AUTH)
  • TLS encryption for secure transmission
  • Extended SMTP (ESMTP) with additional features
Security Considerations:
Use SMTPS or STARTTLS for encryption. Implement SPF, DKIM, and DMARC to prevent email spoofing and improve deliverability.
IMAP
143/993
Internet Message Access Protocol allows email clients to access and manage email messages stored on a remote server. Supports multiple device access.
  • Server-side message storage
  • Multiple folder hierarchy support
  • Partial message downloading
  • Server-side search capabilities
  • Flag and label management
Security Considerations:
Always use IMAPS for encrypted connections. Implement strong authentication and consider OAuth2 for modern applications.
POP3
110/995
Post Office Protocol version 3 downloads email messages from server to client, typically removing them from the server. Simpler than IMAP but less flexible.
  • Download and delete model
  • Single device access pattern
  • Minimal server storage requirements
  • Simple command set
  • Offline email access
Security Considerations:
Use POP3S for secure connections. Consider IMAP for better security and multi-device support in modern environments.
FTP
21
File Transfer Protocol enables file sharing between computers over a network. Uses separate control and data connections for efficient file transfer.
  • Separate control (port 21) and data connections
  • Active and passive transfer modes
  • ASCII and binary transfer types
  • Directory listing and navigation
  • Resume capability for interrupted transfers
Security Considerations:
FTP transmits data in clear text. Use FTPS or SFTP for secure file transfers. Implement proper access controls and monitoring.
SFTP
22
SSH File Transfer Protocol provides secure file transfer over SSH. Encrypts both commands and data, making it the preferred choice for secure file operations.
  • Built on SSH for security
  • Encrypted file transfer and commands
  • Public key authentication support
  • File integrity verification
  • Platform independent
Security Considerations:
Use key-based authentication instead of passwords. Regularly update SSH configurations and monitor access logs for suspicious activity.
FTPS
990/21
FTP Secure adds SSL/TLS encryption to traditional FTP. Available in explicit and implicit modes for securing file transfers while maintaining FTP compatibility.
  • SSL/TLS encryption for FTP
  • Explicit and implicit security modes
  • Certificate-based authentication
  • Backward compatibility with FTP clients
  • Data and control channel encryption
Security Considerations:
Ensure proper certificate validation. Use strong cipher suites and regularly update SSL/TLS configurations.
DNS
53
Domain Name System translates human-readable domain names into IP addresses. Critical infrastructure service that enables user-friendly internet navigation.
  • Hierarchical namespace structure
  • Distributed database system
  • Caching for performance optimization
  • Multiple record types (A, AAAA, MX, CNAME, etc.)
  • Load balancing and failover support
Security Considerations:
Implement DNSSEC for authenticity. Use DNS over HTTPS (DoH) or DNS over TLS (DoT) for privacy. Monitor for DNS poisoning and cache pollution attacks.
DHCP
67/68
Dynamic Host Configuration Protocol automatically assigns IP addresses and network configuration to devices. Simplifies network administration and device connectivity.
  • Automatic IP address assignment
  • Network configuration distribution
  • Lease management and renewal
  • DHCP reservations for specific devices
  • Option sets for additional configuration
Security Considerations:
Secure DHCP servers against rogue DHCP attacks. Use DHCP snooping and implement proper VLAN segmentation.
NTP
123
Network Time Protocol synchronizes computer clocks across networks. Essential for logging, security, and distributed system coordination.
  • Hierarchical time distribution
  • Microsecond precision synchronization
  • Automatic server selection
  • Authentication support
  • Leap second handling
Security Considerations:
Use authenticated NTP to prevent time manipulation attacks. Secure NTP servers and monitor for unusual time changes.
BGP
179
Border Gateway Protocol is the routing protocol of the internet. Manages routing between autonomous systems and enables global internet connectivity.
  • Path vector routing protocol
  • Policy-based routing decisions
  • Loop prevention mechanisms
  • Attribute-based path selection
  • Incremental updates for efficiency
Security Considerations:
Implement BGP security features like RPKI. Monitor for route hijacking and use route filtering to prevent unauthorized announcements.
OSPF
89
Open Shortest Path First is a link-state routing protocol used within autonomous systems. Provides fast convergence and efficient routing.
  • Link-state routing algorithm
  • Hierarchical area design
  • Fast convergence after topology changes
  • VLSM and CIDR support
  • Authentication mechanisms
Security Considerations:
Enable OSPF authentication to prevent routing attacks. Secure area borders and monitor for LSA flooding attacks.
RIP
520
Routing Information Protocol is one of the oldest distance-vector routing protocols. Simple to configure but limited in scalability and convergence speed.
  • Distance-vector algorithm
  • Hop count metric (max 15)
  • Periodic route updates
  • Split horizon loop prevention
  • RIPv2 adds subnet mask support
Security Considerations:
RIP has limited security features. Use RIPv2 with authentication and consider migration to more secure protocols for critical networks.
IPSec
500/4500
Internet Protocol Security provides cryptographic security for IP communications. Supports both tunnel and transport modes for VPN and end-to-end encryption.
  • Authentication Header (AH) protocol
  • Encapsulating Security Payload (ESP)
  • Internet Key Exchange (IKE) for key management
  • Tunnel and transport modes
  • Perfect forward secrecy support
Security Considerations:
Use strong encryption algorithms and regularly update keys. Monitor for configuration errors that could weaken security.
TLS/SSL
443
Transport Layer Security provides cryptographic protocols for secure communication. Widely used for HTTPS, email security, and application-level encryption.
  • Handshake protocol for key exchange
  • Record protocol for data encryption
  • Certificate-based authentication
  • Perfect forward secrecy
  • TLS 1.3 with improved security and performance
Security Considerations:
Use TLS 1.2 or higher with strong cipher suites. Implement proper certificate validation and consider certificate pinning for critical applications.
SSH
22
Secure Shell provides encrypted remote access and secure communication channels. Essential for secure system administration and encrypted tunneling.
  • Strong cryptographic authentication
  • Encrypted command execution
  • Port forwarding and tunneling
  • Public key authentication
  • Agent forwarding for key management
Security Considerations:
Disable password authentication when possible. Use key-based authentication, change default ports, and implement fail2ban for brute force protection.
