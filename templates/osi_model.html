OSI Reference Model

The seven-layer foundation of network communication

The Seven Layers of Network Communication
7
Application Layer
Network services to applications and end-users
🖥️

The Application Layer provides network services directly to end-users and applications. It's the layer where users interact with the network through applications like web browsers, email clients, and file transfer programs.

Common Protocols:
HTTP/HTTPS FTP SMTP POP3 IMAP DNS DHCP SSH
6
Presentation Layer
Data translation, encryption, and compression
🔒

The Presentation Layer transforms data to and from the format required by the Application Layer. It handles encryption, decryption, compression, and data format conversion to ensure data from the sending application can be read by the receiving application.

Key Functions:
SSL/TLS JPEG GIF MPEG ASCII EBCDIC MIME
5
Session Layer
Session management and control
🔗

The Session Layer establishes, manages, and terminates sessions between applications. It controls dialogues and connections between computers, including authentication, authorization, and session restoration.

Session Management:
NetBIOS RPC SQL NFS SMB PPTP
4
Transport Layer
Reliable end-to-end data delivery
🚚

The Transport Layer ensures reliable, error-free data transmission between end systems. It provides error detection and recovery, flow control, and segmentation of large messages into smaller packets for transmission.

Transport Protocols:
TCP UDP SCTP DCCP
3
Network Layer
Routing and logical addressing
🌐

The Network Layer handles routing of data packets between different networks. It determines the best path for data transmission and provides logical addressing through IP addresses, enabling internetwork communication.

Network Protocols:
IP (IPv4/IPv6) ICMP ARP OSPF BGP RIP
2
Data Link Layer
Node-to-node delivery and error detection
🔗

The Data Link Layer provides reliable transfer of data across the physical link. It handles error detection and correction, frame synchronization, and flow control between directly connected nodes.

Data Link Technologies:
Ethernet Wi-Fi (802.11) PPP Frame Relay ATM HDLC
1
Physical Layer
Raw bit transmission over physical medium
⚡

The Physical Layer defines the electrical, mechanical, and functional specifications for activating, maintaining, and deactivating physical connections. It deals with the actual transmission of raw bit streams over a physical medium.

Physical Media:
Copper Cable Fiber Optic Radio Waves Infrared Bluetooth USB

Troubleshooting by OSI Layer

Systematic Network Problem Diagnosis

⚡
Physical Layer Issues

Cable problems, port failures, power issues, signal interference. Check physical connections, LED indicators, and cable integrity.

🔗
Data Link Problems

MAC address conflicts, VLAN misconfigurations, spanning tree issues. Verify switch configurations and frame forwarding.

🌐
Network Layer Issues

IP address conflicts, routing problems, subnet misconfigurations. Check routing tables, IP assignments, and reachability.

🚚
Transport Layer Problems

Port blocking, firewall issues, connection timeouts. Verify port accessibility and connection establishment.

💡 Pro Tip: Start from the Bottom

Always troubleshoot from Layer 1 (Physical) upward. Many network issues are caused by simple physical problems like loose cables or power failures. Verify each layer works before moving to the next.
